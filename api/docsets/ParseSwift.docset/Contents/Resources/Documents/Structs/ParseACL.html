ParseACL

public struct ParseACL : Codable, Equatable
extension ParseACL: CustomDebugStringConvertible

Objects that conform to the ACL protocol are used to control which users can access or modify a particular ParseObject. Each ParseObject can have its own ACL. You can grant read and write permissions separately to specific users, to groups of users that belong to roles, or you can grant permissions to “the public” so that, for example, any user could read a particular object but only a particular set of users could write to that object.

  • setDefaultACL(_:withAccessForCurrentUser:)

    Sets a default ACL that will be applied to all instances of ParseObject when they are created.

    This value will be copied and used as a template for the creation of new ACLs, so changes to the instance after this method has been called will not be reflected in new instance of ParseObject.

    • If false, the provided acl will be used without modification.

    • If acl is nil, this value is ignored.

    Declaration
